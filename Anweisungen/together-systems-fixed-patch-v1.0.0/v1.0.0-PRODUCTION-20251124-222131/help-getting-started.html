1. Portal als App installieren index.html · manifest-portal.webmanifest · sw.js

Desktop (Chrome / Edge):

  • Öffne https://ts-portal.pages.dev/index.html.
  • Achte auf das kleine „Installieren“-Symbol in der Adressleiste oder wähle im Browser‑Menü „Diese Seite installieren“.
  • Bestätige die Installation – danach findest du das Portal als eigenes Programm‑Fenster mit Icon.

Android‑Handy (Chrome):

  • Öffne dieselbe Adresse im Chrome‑Browser.
  • Über das Menü „Zum Startbildschirm hinzufügen“ / „App installieren“ auswählen.
  • Es entsteht ein Icon auf dem Homescreen – Start wie eine normale App, inklusive Offline‑Funktion.
📸 Screenshot‑Platzhalter: Hier kannst du später Bilder einfügen, z. B. „Installieren‑Icon in Chrome“ und „Portal‑Icon auf dem Desktop/Homescreen“.

2. Offline‑Manifest einmalig benutzen manifest-forum.html

Download aus dem Portal:

  • Gehe in index.html zum Abschnitt „Manifest‑Forum: Teilnahme für Einsteiger (Download + Erklärung)”.
  • Klicke auf den Button „Manifest‑Forum herunterladen“ – es wird manifest-forum.html gespeichert.
  • Verschiebe diese Datei an einen festen Ort (z. B. Desktop oder einen speziellen Ordner) und benutze immer genau diese Kopie.

Verwendung:

  • Doppelklick auf die Datei → Offline‑Forum öffnet sich im Browser.
  • Beiträge erstellen, Reaktionen, Kommentare, Export/Import – alles passiert lokal im Browser‑Speicher.
  • Mit „Offline‑Kopie speichern“ kannst du eine Sicherung der Datei erstellen.
📸 Screenshot‑Idee: Ausschnitt des Download‑Bereichs in index.html und Ansicht von manifest-forum.html mit Formular „Beitrag erstellen“.

3. Verifizierung & Auto‑Connect manifest-forum.html → manifest-portal.html

Brücke vom Offline‑Forum ins Online‑Portal:

  • Im Offline‑Forum gibt es (rechts) den Button „Portal öffnen (verifiziert)“.
  • Beim Klick wird eine URL wie manifest-portal.html#mot=…&uid=…&ts=… geöffnet – damit übergibt die Datei ein Token an das Online‑Portal.
  • Im Portal liest der Code diesen Hash aus, markiert den Nutzer als „verifiziert“ und kann Präsenz‑ und Matching‑Funktionen starten.

Was du sehen solltest:

  • Oben im Portal‑Header: „Manifest of Thinkers – Portal“.
  • Im Verifizierungs‑Bereich wird das Token‑Feld ausgefüllt und der Status‑Text wechselt von „Warte auf Verbindungsaufbau …“ zu einer Verifizierungs‑Nachricht (wenn Backend aktiv ist).
  • Im Bereich „Direktverbinden (Auto‑Connect)“ kannst du ein gemeinsames Stichwort (pair code) eingeben, um automatisch mit einem zweiten Thinker in einen Live‑Raum verbunden zu werden.

Plug‑and‑Play‑Gefühl („Schwimmbad‑Modus“):

  • Du musst keine Hashes oder Signaturen verstehen – der Bereich „Token‑URL generieren (No‑Code)“ baut die komplette Link‑Adresse für dich.
  • Du kopierst nur die fertige URL oder klickst im Offline‑Manifest auf „Portal öffnen (verifiziert)“ – der Rest passiert automatisch.

5. Häufige Probleme (Desktop & Mobil)

  • „Installieren“-Dialog erscheint nicht:
    • Sicherstellen, dass du /index.html über HTTPS öffnest (z. B. https://ts-portal.pages.dev/).
    • Im Browser‑Cache alte Versionen von manifest-portal.webmanifest und sw.js ggf. löschen (Hard‑Reload / Cache leeren).
  • Verifizierung bleibt hängen:
    • Prüfen, ob der Link aus dem Offline‑Manifest wirklich mit #mot=…&ts=… endet (Adresse oben im Browser ansehen).
    • Wenn das Backend (Cloudflare Functions) nicht läuft, wird der Live‑Status nur teilweise aktualisiert – das Lesen des Feeds funktioniert aber trotzdem.
  • Seite landet im Download‑Ordner:
    • Das ist beim manifest-forum.html bewusst so – dort bitte nach dem ersten Download an einen festen Platz verschieben (z. B. Desktop).
    • Für das tägliche Arbeiten ist das Portal als PWA gedacht; dort wird nichts mehr als neue Datei heruntergeladen.

Weitere technische Hintergrund‑Infos findest du in der Dokumentation im Projekt, z. B. COMMUNICATION-HUB-ARCHITECTURE.md und DEVELOPMENT-GUIDE-TEL-PORTAL.md.

6. Glossar für Einsteiger

  • PWA (Progressive Web App): Webseite, die sich wie eine installierbare App verhält (Icon, Vollbild, Offline‑Fähigkeit).
  • Web App Manifest: Kleine JSON‑Datei (manifest-portal.webmanifest), in der Name, Icons, Start‑URL und Farben der App beschrieben sind.
  • Service Worker: Hintergrund‑Script (sw.js), das Dateien cacht und Offline‑Betrieb ermöglicht.
  • Offline‑Manifest: Die Datei manifest-forum.html, die komplett im Browser läuft und Daten lokal speichert.
  • Playwright: Test‑Werkzeug, das den Browser ferngesteuert bedient (Klicks, Texteingaben, Prüfungen) und so automatisierte End‑to‑End‑Tests ermöglicht.

Für mehr technische Tiefe kannst du gezielt nach Wörtern wie „PWA“, „Service Worker“ oder „WebSocket“ suchen – sowohl in dieser Hilfe als auch im restlichen Projekt.
